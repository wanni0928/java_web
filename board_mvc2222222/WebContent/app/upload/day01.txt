JSP(Java Server Page)
	HTML을 중심으로 자바와 같이 연동하여 사용하는 웹 언어이다.
	HTML코드 안에 JAVA코드를 작성할 수 있다.

서버
	사용자의 요청에 맞는 서비스를 제공해주는 것

		요청(request)
		→→→→→→→→→
	클라이언트		서버
		←←←←←←←←←
			응답(response)

httpd
	클라이언트의 페이지 요청을 인식하는 프로그램이며 웹 서버에 요청을 알린다.

웹(Web)
	페이지 요청과 응답이 일어나는 장소
	인터넷에 연결된 컴퓨터들을 통해 사람들이 정보를 공유할 수 있는 정보공간.

웹 서버(http)
	사용자의 요청이 정적 데이터인지 동적 데이터인지 판단한다.
	정적 데이터일 경우 이미 준비된 HTML문서를 그대로 응답해 주며,
	동적 데이터라면 웹 컨테이너(서블릿 컨테이너)에 요청을 보낸다.

웹 컨테이너(서블릿 컨테이너)
	사용자의 요청이 동적 데이터일 경우 JSP, 서블릿으로 연산 및 제어, DB에 접근
	등을 통해 정제된 데이터(결과)가 완성되면 웹 서버를 거쳐 응답해 준다.

WAS(Web Application Server)
	동적 데이터를 처리할 서블릿을 메모리에 할당하며, web.xml을 참조 후
	해당 서블릿에 대한 Thread 생성, 서블릿 요청과 서블릿 응답 객체 생성 후
	웹 서버가 요청한 동적 데이터를 서블릿에 전달한다.

서블릿(Servlet)
	JAVA코드 안에 HTML코드를 작성할 수 있는 JAVA 프로그램이다.
	Thread에 의해 서블릿에 있는 service()메소드가 호출된다.
	전송 방식 요청에 맞게 doGet() 또는 doPost()메소드가 호출된다.
	-doGet(HttpServletRequest req, HttpServletResponse res){}
	-doPost(HttpServletRequest req, HttpServletResponse res){}

WAS는 Response객체를 HttpResponse형태(정적)로 바꿔서 웹 서버에 전달하고
생성된 Thread를 종료한다. 그리고 HttpServletRequest와 HttpServletResponse객체를
제거한다.

스크립트 태그
	HTML코드에 자바 코드를 넣어 프로그램이 수행하는 기능을 구현할 수 있다.

	1. 선언문(declaration) : <%! %>
		자바 변수나 메소드를 정의하는 데 사용하는 태그
	2. 스크립틀릿(scriptlet) : <% %>
		자바 변수 선언 및 자바 로직 코드를 작성하는 데 사용하는 태그
	3. 표현문(expression) : <%= %>
		변수, 계산식, 메소드 호출 결과를 나타내주는 태그. 타입은 문자열이다.

디렉티브 태그
	<%@ page %> : <%와 @사이에 공백이 없어야 한다.
	현재 JSP페이지에 대한 정보를 설정하는 태그이다.
	되도록 페이지 최상단에 선언한다.	
	
	0. 속성명(기본값)
	1. language(java)		: 사용할 프로그래밍 언어
	2. contentType(text/html)	: 생성할 문서의 콘텐츠 유형
	3. pageEncoding(ISO-8859-1)	: 문자 인코딩 설정
	4. import			: 사용할 자바 클래스 설정
	5. session(true)		: 세션 사용 여부 설정
	6. info			: 페이지에 대한 설명 작성
	7. errorPage		: 오류 페이지 설정
	8. isErrorPage(false)	: 오류페이지 여부 설정

	import 속성을 제외한 속성은 한 번씩만 설정할 수 있다.

include 디렉티브 태그
	<%@ include file="파일명">
	현재 JSP페이지의 특정영역에 외부 파일의 내용을 포함하는 태그이다.
	보통 header와 footer는 대부분의 페이지에 동일한 내용으로 작성기 때문에
	유지보수 및 편의를 위하여 외부 파일로 만든 후 include하여 사용한다.
	이러한 것을 모듈화라고 한다.

액션 태그
	서버나 클라이언트에게 어떤 행동을 하도록 명령하는 태그
	페이지와 페이지 사이를 제어하거나 다른 페이지의 실행 결과 내용을
	현재 페이지에 포함하거나 자바빈즈(객체) 등의 다양한 기능을 제공한다.
	액션 태그는 XML형식인 <jsp : />를 사용한다.

	1. forward(<jsp:forward />)
		다른 페이지로의 이동, 페이지 흐름 제어.
	2. include(<jsp:include />)
		외부 페이지의 내용을 포함하거나 페이지 모듈화.
	3. param(<jsp:param />)
		현재 페이지에서 다른 페이지에 정보를 전달한다.
	4. useBean(<jsp:useBean />)
		JAVA파일에 있는 객체를 가지고 온다.
	5. setProperty(<jsp:setProperty />)
		자바빈즈의 프로퍼티 값을 설정한다.
	6. getProperty(<jsp:getProperty />)
		자바빈즈의 프로퍼티 값을 가지고 온다.
